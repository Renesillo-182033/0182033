Objetivo General

Aplicar el modelo CVSS v3.1 para evaluar la gravedad de vulnerabilidades reales, interpretando correctamente sus métricas base y justificando la priorización del riesgo en un contexto organizacional.

Objetivos Específicos

Identificar y comprender las métricas base de CVSS:

AV — Attack Vector

AC — Attack Complexity

PR — Privileges Required

UI — User Interaction

S — Scope

C — Confidentiality

I — Integrity

A — Availability

Construir correctamente una cadena vector CVSS a partir de un escenario técnico.

Calcular e interpretar la puntuación de riesgo para priorizar acciones de mitigación.

Escenario

Una organización detecta una vulnerabilidad en un sistema web interno con las siguientes características:

1. Vector de ataque

Puede explotarse a través de la red.

2. Complejidad del ataque

La explotación requiere baja complejidad.

3. Privilegios requeridos

Se necesitan privilegios elevados.

4. Interacción del usuario

No requiere interacción del usuario.

5. Alcance

No cambia el alcance.

6. Impacto

Confidencialidad: Bajo.

Integridad: Bajo.

Disponibilidad: Bajo.

Cadena de Vector

CVSS:3.1/AV:N/AC:L/PR:H/UI:N/S:U/C:L/I:L/A:L

Interpretación Técnica

La vulnerabilidad sigue siendo relevante porque un atacante que ya tenga acceso administrativo o credenciales privilegiadas puede explotarla fácilmente.

El atacante que podría aprovecharse sería un usuario interno malicioso, un administrador comprometido o un atacante que previamente obtuvo acceso privilegiado mediante robo de credenciales o el uso de otras técnicas.

Que el impacto sea bajo en confidencialidad, integridad y disponibilidad significa que la vulnerabilidad genera afectaciones limitadas.

Por ejemplo, la información expuesta es reducida, las modificaciones al sistema no son graves y la disponibilidad del servicio solo se afecta mínimamente.

Cálculo de Puntuación

Puntuación Base

4.7

Categoría

Media (4.0 – 6.9)

La vulnerabilidad se clasifica como de riesgo medio según la escala CVSS v3.1.
