Parte 01. Comprensión del Modelo

Se completó la tabla conceptual definiendo cada elemento del Modelo Diamante y asignando un ejemplo práctico correspondiente a escenarios reales de ciberseguridad.

Elemento

Descripción

Ejemplo práctico

Adversario

Se define como el actor de amenaza, individuo o grupo responsable de dirigir y ejecutar la intrusión cibernética para alcanzar un objetivo específico.

Se identifica a un grupo APT o a un atacante externo motivado por la exfiltración de información confidencial.

Capacidad

Se refiere al conjunto de herramientas, técnicas, tácticas y procedimientos utilizados por el adversario.

Se empleó un payload malicioso y técnicas de inyección SQL para extraer registros de bases de datos.

Infraestructura

Se describe como la estructura física y lógica utilizada para controlar capacidades y mantener comunicaciones.

Se observó un servidor VPS configurado como C2 junto con dominios utilizados para phishing.

Víctima

Es el objetivo principal del ataque, pudiendo ser personas, organizaciones o activos de información.

El objetivo comprometido fue un servidor interno y credenciales administrativas.

Análisis Detallado del Escenario

1. ¿Quién es el adversario en este escenario?

Se determinó que el adversario es un actor externo con capacidad de administrar dominios y servidores utilizados para campañas de malware.

2. ¿Cuál es la capacidad utilizada?

La capacidad principal consistió en malware avanzado capaz de evadir detecciones básicas y mantener conexiones persistentes.

3. ¿Qué tipo de infraestructura se emplea?

Se identificó infraestructura híbrida conformada por dominios específicos y direcciones IP externas utilizadas como nodos C2.

4. ¿Quién es la víctima primaria?

La víctima primaria fue la organización afectada y los equipos donde se detectó inicialmente el comportamiento anómalo.

5. ¿Existe evidencia de movimiento lateral?

Sí. Los registros de red mostraron múltiples hosts internos conectándose a las mismas direcciones IP externas.

Justificación Técnica de la Clasificación

Envío de Phishing

Corresponde a la fase de Entrega debido a que representa el mecanismo utilizado para trasladar el payload malicioso hacia la víctima.

Ejecución del Malware

Se clasifica dentro de las fases de Explotación e Instalación ya que el malware logra persistencia dentro del sistema.

Conexión a C2

Se relaciona directamente con la fase de Comando y Control permitiendo administración remota y exfiltración de datos.

Detección de Malware

Aunque es un evento defensivo, las anomalías detectadas se originaron durante fases avanzadas de la intrusión.
